# Launchbase

A full-stack monorepo template: a FastAPI backend and a Next.js SSG-first
frontend, with shared tooling wired for monorepo development. Opinionated
defaults so you can skip the wiring and start building.

## Structure

| Folder      | What it is                                                              |
| ----------- | ----------------------------------------------------------------------- |
| `api/`      | FastAPI backend — async SQLAlchemy, Postgres, Alembic, JWT auth, pytest |
| `platform/` | Next.js frontend — SSG-first (Jamstack), Tailwind, shadcn/ui            |
| `admin/`    | Next.js admin console — SSG-first, user management on the API `/admin` surface |
| `site/`     | Next.js landing page — SSG-first, API-less, full shadcn/ui component set |

Each folder has its own README with setup and workflow details:
[api/README.md](api/README.md), [platform/README.md](platform/README.md),
[site/README.md](site/README.md).

## Requirements

- Python 3.12 and [uv](https://docs.astral.sh/uv/)
- Node.js 24+ (use `nvm use` inside `platform/` to pick up `.nvmrc`)
- Postgres 16

## Quick start

```bash
git clone <repo-url> && cd launchbase

# API
cd api
uv sync
cp .env.example .env                     # then edit values
createdb launchbase
uv run alembic upgrade head
uv run fastapi dev                       # http://localhost:8000
cd ..

# Platform
cd platform
npm install
cp .env.example .env.local               # then edit values
make run                                 # http://localhost:3001
cd ..

# Site (landing page)
cd site
npm install
make run                                 # http://localhost:3000
```

## API SDK generation

`platform/src/api/` holds a typed client (TanStack Query hooks + zod schemas)
generated by [orval](https://orval.dev) from the API's OpenAPI spec. With the
API dev server running, regenerate it after backend changes:

```bash
cd platform && npm run generate
```

## Dev session

[tmuxp](https://tmuxp.git-pull.com/) is a tmux session manager that builds
preconfigured sessions from a YAML file. Run `tmuxp load .` from the repo root
to start the dev session (API server + platform dev server).

## Git hooks

Git hooks for the whole monorepo are managed by
[pre-commit](https://pre-commit.com) via `.pre-commit-config.yaml` at the repo
root: generic file checks everywhere, ruff/pyright for `api/`, and
prettier/eslint/tsc for `platform/`. Install once after cloning:

```bash
uv run --project api pre-commit install
```

Run all hooks manually with:

```bash
uv run --project api pre-commit run --all-files
```
